# Temperature-dependent Jiles-Atherton model of gadolinium

Code and data for the Jiles-Atherton (JA) analysis of magnetic hysteresis in
polycrystalline gadolinium across its ferromagnetic range: twelve major B(H)
loops measured from -38 to +15 degrees C (the Curie point of the sample is
Tc = 18.3 degrees C = 291.5 K), described by one set of smooth, physically
interpretable temperature laws, with a GUM-compliant uncertainty assessment.
The repository accompanies the manuscript "Magnetization scaling and collapse
of the pinning energy scale in gadolinium approaching the Curie point"
(M. Nowicki, R. Szewczyk, A. Kolano-Burian, T. Charubin), submitted to
Physica B: Condensed Matter.

Measurement data: Dr. Michal Nowicki. Author of the analysis code: Roman
Szewczyk. MIT License (see `LICENSE`).

## Results

A 12-parameter global model reproduces all twelve major loops with a mean
adjusted R^2 of 0.99932 +/- 0.00022 (every loop above 0.99; mean 0.99923 in
the demanding -15 to +10 degrees C range near Tc). The root-mean-square
flux-density residual per loop is 20-32 mT, 27 mT on average, which is
0.8-1.4 % of each loop's peak-to-peak span.

```
Ms(T_K) = 2.139e6 * (1 - T_K/291.48)^0.281        [A/m]  critical power law
a(t)    = -0.4341*t^2 + 87.18*t + 7447.7          [A/m]  monotonic increase
k(t)    =  0.0824*t^2 -  1.835*t +   78.2         [A/m]  monotonic decrease (depinning near Tc)
c(t)    =  2.416e-4*t^2 + 5.333e-3*t + 0.1593     [-]    single minimum near -11 C
alpha   =  0                                             (t in degrees C, T_K in K)
```

Combined standard uncertainties of the order-parameter quantities:
Ms0 = 2.14 +/- 0.08 MA/m (3.6 %), Tc = 291.5 +/- 0.7 K (0.25 %),
beta = 0.281 +/- 0.019 (6.6 %). The global fit is well conditioned (condition
number of the scaled normal matrix approx. 1.1e4) and determines all four
temperature laws. The parameters are well determined but mutually dependent
(correlations a-Ms approx. +0.92, a-k approx. -0.35, k-c approx. +0.18), so
they are read jointly rather than in isolation. The decreasing trend of k(T)
toward Tc survives in more than 99 % of the Monte Carlo trials, and the
measurement system is not the factor limiting the analysis.

## Reproducing the analysis

All scripts resolve their paths through `02_scripts/paths.py`, so they can be
run from any working directory. Run the two fit stages in order; Stage B reads
the Stage A output to seed its initial guess (with hard-coded physical
defaults as a fallback), so the order matters for the quality of the starting
point.

```bash
cd 02_scripts
python3 identify_per_temperature.py     # Stage A (minutes; parallel). Optional args: [start_idx] [end_idx]
python3 global_physical_ja_fit.py       # Stage B (slow; parallel global search)
python3 plot_physical_model.py          # loop + parameter figures for both fits
python3 plot_paper_figures.py           # a(T)/effective-moment and k(T)/c(T) figures
python3 beta_window_crosscheck.py       # robustness of beta to fit window and assumed Tc
python3 alpha_crosscheck.py             # robustness of the alpha = 0 assumption
python3 export_results.py               # CSV and LaTeX parameter tables
```

Then the uncertainty assessment, in this order:

```bash
cd ../04_uncertainty_assessment
python3 correlation_analysis.py         # empirical covariance of (a, k, c, Ms)
python3 propagate_calibration.py        # Component A (~25 s, one global Jacobian)
python3 propagate_identifiability.py    # Component B (1e5-trial correlated Monte Carlo, ~2 min)
python3 combine_and_report.py           # summary JSON/CSV and LaTeX tables
python3 plot_uncertainty.py             # Ms(T) power law with 95 % band
```

The uncertainty scripts read `../03_results` and `../02_scripts` and never
modify them; their outputs go to `04_uncertainty_assessment/results/`.
Committed results in `03_results/json/` and
`04_uncertainty_assessment/results/json/` allow every plotting, export and
uncertainty script to run without repeating the slow Stage B fit.

## Method in brief

Stage A fits each loop independently for (a, k, c, Ms) at alpha = 0. These
fits reach adjusted R^2 approx. 0.999 at every temperature, but a single loop
does not determine k and c uniquely near the Curie point, so their per-loop
values scatter. Stage A therefore serves as a diagnostic and as a source of
priors.

Stage B removes that degeneracy by fitting all twelve loops simultaneously
with the temperature dependence of every parameter constrained to a smooth
law: Ms as the critical power law Ms0*(1 - T_K/Tc)^beta, and a, k, c as
quadratics in t (degrees C), each pinned by its values at the three anchor
temperatures [-38, -11.5, +15]. The parameter vector is
x = [Ms0, beta, Tc, a/k/c at the three anchors], twelve numbers in total. The
objective is the mean over loops of (1 - R^2), which weights every loop
equally; a plain sum of squared errors would let the cold, high-Ms loops
dominate and under-fit near Tc. The search runs differential evolution
(seed 42, parallel workers) on a 500-point subsample of each loop, polishes
with Nelder-Mead, and reports final metrics on the full loops.

The JA differential equation is integrated per monotonic field segment with
an adaptive RK45 solver, which avoids the numerical instability that
fixed-step integration shows on the cold, high-Ms loops. The Langevin
function and its derivative are evaluated from their Taylor series below
|He/a| = 1e-3, where the closed forms would lose precision to cancellation.
The per-loop initial magnetisation is taken from the data as
M0 = B[0]/mu0 - H[0]. Unphysical parameter sets receive a finite 1e30
penalty so the optimiser can recover from them.

Two dedicated cross-checks bound the main modelling choices. The window/Tc
scan (`beta_window_crosscheck.py`) shows beta stays in 0.28-0.30 for every
fit window down to tau_min approx. 0.012, so the low apparent exponent is not
a finite-window artefact; it is instead controlled by the assumed Tc, rising
to about 0.31 when Tc is held at the single-crystal bulk value 293 K. The
alpha profile (`alpha_crosscheck.py`) re-optimises (a, k, c, Ms) for every
loop at fixed alpha up to 3e-3 and finds R^2 gains only in the fourth decimal
place and power-law shifts within the quoted uncertainties, which justifies
fixing alpha = 0.

## Uncertainty assessment in brief

Two components are propagated into the global model and combined as
u_c = sqrt(u_cal^2 + u_idn^2):

Component A, calibration (Type B): a single gain factor per channel (0.1 %
on B, 0.01 % on H, fully correlated across all loops) propagated through the
well-posed global fit by the GUM law of propagation of uncertainty. It
rescales the loop amplitudes without changing their shape and therefore lands
almost entirely in Ms0 (approx. 0.08 %).

Component B, identifiability (Type A, dominant): a correlated Monte Carlo
with 1e5 trials (JCGM 101). The four JA parameters are not independent; their
joint covariance is estimated empirically from log-residuals of the
per-temperature fits about the smooth global laws, and each trial perturbs
the twelve per-loop parameter sets by a multivariate-lognormal factor and
re-fits the Ms(T) power law.

Propagation is done at the global level, never by re-fitting single loops:
single-loop re-fitting moves (a, k, c) by tens of percent along the
degenerate direction at constant fit quality, which is exactly the
ill-conditioning the global fit removes. Read
`04_uncertainty_assessment/uncertainty_method.md` before touching this
module.

## Using the fitted model

```python
import json, numpy as np

c = json.load(open('03_results/json/physical_model_coeffs.json'))

def ja_params(t):
    T_K = t + 273.15
    Ms = c['Ms0'] * (1 - T_K / c['Tc']) ** c['beta']
    a = np.polyval(c['a_coeffs'], t)
    k = np.polyval(c['k_coeffs'], t)
    rev = np.polyval(c['c_coeffs'], t)
    return a, k, rev, Ms        # alpha = 0
```

## Data

`gd_processed.npz` (load with `np.load(..., allow_pickle=True)`; the arrays
are ragged) holds `temperatures` and the pairs `major_H/major_B`,
`mid_H/mid_B`, `minor_H/minor_B`. Fields range from 400 to 55000 A/m
(25 amplitudes) at 3 Hz. The present analysis uses the major loops (peak
field 55 kA/m). The `allow_pickle=True` load is intentional and applies only
to this trusted local file; do not extend it to untrusted inputs.

## Verification

There is no automated test framework. Verification means re-running the
pipeline and reproducing the headline metrics (mean adjusted R^2 approx.
0.99932, all twelve loops above 0.99), running the two cross-check scripts,
and confirming the JSON outputs stay consistent with the numbers quoted
above. `CODE_REVIEW.md` records the step-by-step check performed for this
final version.
